# Float Ball Domain

Owns trusted browser-to-Companion binding and system float ball orchestration.

- A browser session can only bind to a Companion after a nonce challenge is generated by OpenFocus and returned by a local Companion through the existing gRPC stream.
- The Companion is selected for system float ball only when it is paired, online, bound to the browser session, and declares `system_float_ball` or a namespaced variant such as `system_float_ball.macos`.
- The web UI always exposes the attention inbox from the navigation bar. The old
  fixed page-level floating bubble is not used as a fallback.
- The navigation inbox can request/stop the system float ball. A successful start
  response means the system helper has acknowledged that its window is visible.
